Programmer ? Mais c'est très simple !

Je vous présente deux (à terme trois) livres pour apprendre à programmer. Ces ouvrages ne sont pour l'instant qu'en version préliminaire et sont encore loin d'être achevés. Ils sont cependant suffisamment avancés pour que je les présente ici (le mot de passe est secret).

 

  • Le tome 1 intitulé « Apprenez à programmer en JavaScript avec Node.js » se propose de vous apprendre la programmation. Le cours s'appuie sur le langage JavaScript, plus précisément sur sa version serveur Node.js. C'est un cours d'initiation à la programmation, destiné aux débutants, qui n'a pas vocation à être exhausitf.
  • Le tome 2 intitulé « Apprenez à développer une application WEB multi-pages » se propose de vous apprendre à développer une application WEB de type multi-pages. Ce type d'application consiste à développer un serveur qui s'utilise via un navigateur. Attention : il est conseillé de posséder un minimum de connaissance des langages HTML et CSS, même si les principes fondamentaux sont rappelés dans le cours.
  • Le tome 3 sera intitulé « Apprenez à développer une application WEB single-page (SPA)» et se proposera de vous apprendre à développer une application WEB de type single-page, encore appelée SPA pour Single Page Application. Cette techologie utilise les possibilités des navigateurs modernes, en particulier la possibilité d'exécuter du code JavaScript à l'intérieur même du navigateur et non plus uniquement dans le serveur.

Apprendre à programmer avec JavaScript/Nodejs ? Et pourquoi pas !

Beaucoup d'informaticiens ont découvert la programmation avec le langage C ou sa variante objet C++. Ce langage est aujourd'hui encore largement enseigné pour faire découvrir la programmation en école d'ingénieur ou à l'université. Créé dans les années 1970, le C est devenu populaire avec UNIX dans les années 1980. Il est encore largement utilisé et irremplaçable dans certains contextes. Mais des langages plus modernes sont apparus depuis et ont supplanté le C pour certains types de développement comme par exemple les applications WEB. Il est probable qu'à terme le C perde son statut de langage d'initiation. Cet ouvrage en fait le pari, et opte pour une découverte de la programmation avec le JavaScript.

JavaScript est souvent largement critiqué par certains « experts » : lui sont reprochés en vrac de ne pas être consistant, d'être trop permissif, insuffisamment typé, etc. En son temps, C était loin de faire l'unanimité : dans les années 1980, beaucoup prédisaient à terme une suprématie du Pascal, langage consistant, non permissif, typé, etc. Ce langage est aujourd'hui quasiment oublié...

Les experts informatiques sont comme les critiques de cinéma : ce ne sont pas eux qui font ou défont les succès. Dans le monde informatique, libéral et complexe, le succès d'une technologie dépend avant tout de sa communauté d'utilisateurs. Seule une communauté nombreuse et active est gage de succès et de pérennité pour une technologie : c'est le cas de JavaScript, et en particulier de son implémentation serveur, Node. Par ailleurs, l'adoption massive de cette technologie depuis le milieu des années 2000, et qui ne se dément pas, démontre que cet engouement n'est pas qu'une mode passagère.

A qui s'adressent ces livres

Ce cours s'adresse en premier lieu à ceux qui veulent découvrir le développement d'applications, éventuellement après un premier vernis en JavaScript ou dans un autre langage.

C'est un cours d'INITIATION à la programmation avec JavaScript et Node.js. Mais attention : ce n'est pas un cours pour APPRENDRE JavaScript ou Node.js.

Ce n'est pas une référence non plus : la « bible » du JavaScript, JavaScript : The Definive Guide par David Flanagan aux éditions O'REILLY, fait près de 1100 pages ! En effet, JavaScript est un langage qui peut sembler simple au premier abord mais qui s'avère en réalité très complexe dès que l'on s'y attarde vraiment, ce qui explique au passage qu'il existe très peu de bons ouvrages sur ce langage (quelques ressources dignes d'intérêt sont référencées en annexe). Je n'ai pas la volonté d'être exhaustif : je laisse volontairement de côté certaines fonctionnalités ou particularités complexes du langage JavaScript. Quand le langage offre plusieurs manières de procéder, je n'en présente souvent qu'une seule. L'apprentissage en général et celui de la programmation en particulier doit parfois se contenter de n'aborder que l'essentiel, au risque de noyer le débutant. C'est le point de vue adopté et assumé dans cet ouvrage.

Devenu plus autonome, vous pourrez vous tourner vers d'autres sources pour compléter vos connaissances, et notamment aborder toutes les subtilités du JavaScript qui en font tout son intérêt. Vous pourrez également vous lancer à la découverte de nouveaux langages.
